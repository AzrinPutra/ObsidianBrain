# CSIT127 (SP425) Networks and Communications


### Network
A network facilitates the sharing, transmission, and management of information. Among the various types of networks, the three most common are the Internet, Intranet, and Extranet.

Internet
The Internet is the world’s largest public network, connecting millions of computers, devices, and servers across the globe. It has no single owner and functions through cooperation between Internet Service Providers (ISPs), organizations, and network infrastructures.

Intranet
An Intranet is a private network designed for internal use within an organization. It leverages the same technologies as the Internet (TCP/IP, HTTP, web browsers) but restricts access only to authorized employees.

Extranet
An Extranet is a controlled extension of an Intranet that allows external stakeholders such as suppliers, clients, or business partners to access certain internal resources. It combines the security of an Intranet with the connectivity of the Internet.

### Types of networks

LAN (Local Area Network)
MAN (Metropolitan Area Network)
WAN (Wide Area Network)
PAN (Personal Area Network)


|**Network Type**|**Full Form**|**Coverage Area**|**Examples / Use Cases**|
|---|---|---|---|
|**LAN**|Local Area Network|Covers a small area like a room, office, or building|Home Wi-Fi, school computer lab, office networks|
|**MAN**|Metropolitan Area Network|Covers a city or large campus|City-wide Wi-Fi, university campus network, cable TV networks|
|**WAN**|Wide Area Network|Covers large geographic areas (countries or continents)|The Internet, corporate branch connections across countries|
|**PAN**|Personal Area Network|Covers very short range (few meters, around a person)|Bluetooth devices, hotspot tethering, smartwatch syncing|

Hub vs Switch vs Router


|**Feature**|**Hub**|**Switch**|**Router**|
|---|---|---|---|
|**OSI Layer**|Physical Layer (Layer 1)|Data Link Layer (Layer 2)|Network Layer (Layer 3)|
|**Working Principle**|Works on broadcasting|Works on **MAC address**|Works on **IP address**|
|**Operation**|Multiport repeater: signal at one port is sent to all ports|Receives message and forwards only to the intended device|Reads packet headers and forwards to the correct network/destination|
|**Intelligence**|Not intelligent (just amplifies signals)|Intelligent (inspects MAC address to forward data)|Highly intelligent (analyzes IP address, builds routing tables, supports filtering/encapsulation)|
|**Networks Required**|At least 1 network|At least 1 network|Requires at least 2 networks|
|**Cost**|Cheapest|More expensive than hub|Most expensive|
|**Speed**|Original hubs: 10 Mbps; modern hubs: 100 Mbps|10 Mbps to 100 Mbps|Wired up to 100 Mbps; Wireless ~1–10 Mbps|
|**Usage**|Used in LANs|Used in LANs|Used in LANs, MANs, WANs|

### FDMA vs TDMA vs CDMA


|**Aspect**|**FDMA** (Frequency Division Multiple Access)|**TDMA** (Time Division Multiple Access)|**CDMA** (Code Division Multiple Access)|
|---|---|---|---|
|**Full Form**|Frequency Division Multiple Access|Time Division Multiple Access|Code Division Multiple Access|
|**Basic Principle**|Bandwidth is divided among different stations.|Time of satellite transponder is divided among different stations.|Both bandwidth and time are shared using unique codes.|
|**Codeword Requirement**|No codeword required.|No codeword required.|Codeword is necessary.|
|**Guard Requirement**|Requires **guard bands** between adjacent channels.|Requires **guard time** between adjacent time slots.|Requires both **guard bands** and **guard time**.|
|**Synchronization**|Not required.|Required.|Not required.|
|**Data Rate**|Low data rate.|Medium data rate.|High data rate.|
|**Mode of Data Transfer**|Continuous signal.|Signal in bursts.|Digital signal.|
|**Flexibility**|Little flexibility.|Moderate flexibility.|Highly flexible.|
|**Conclusion**|Divides frequency for each user.|Divides time slots for each user.|Uses unique codes so all users share the same frequency at the same time.|
|**Usage Example**|Analog systems, satellite links.|GSM (2G) mobile networks.|3G and later cellular systems.|

In conclusion, ARP helps computers find each other's **[physical addresses](https://www.geeksforgeeks.org/operating-systems/logical-and-physical-address-in-operating-system/)** on a network so they can communicate effectively. ARP (Address Resolution Protocol) is like a translator for computers on a network. When one computer wants to talk to another, it needs to know the other computer's physical address (MAC address). But all it has is the other computer's IP address (like its home address). So, ARP steps in and asks, "Hey, who has this IP address?" The computer with that **[IP address](https://www.geeksforgeeks.org/computer-science-fundamentals/what-is-an-ip-address/)** responds with its **[MAC address](https://www.geeksforgeeks.org/computer-networks/mac-address-in-computer-network/)**, and then they can chat.
## Relationship Between ARP , DNS and DHCP
As was previously noted, IP addresses are dynamic by design since doing so protects users' privacy and security. IP address changes, though, shouldn't happen at random. An IP address should be assigned according to rules from a predetermined range of numbers that are available in a particular network. By doing this, problems like two machines getting the same IP address are avoided.
The Dynamic Host Configuration Protocol, or **[DHCP](https://www.geeksforgeeks.org/computer-networks/dynamic-host-configuration-protocol-dhcp/)**, is the name given to the regulations. Because IP addresses are required to do an internet search, they are significant as computer identities. Users utilise alphabetical names while searching for a domain name or Uniform Resource Locator (URL).
Computers, on the other hand, link a domain name to a server using the numeric IP address. In order to link the two, an IP address is converted from a bewildering string of digits into a more legible, intelligible domain name by use of a Domain Name System (**[DNS](https://www.geeksforgeeks.org/computer-networks/domain-name-system-dns-in-application-layer/)**) server, and vice versa.

### TCP
- Connection based
- TCP layer creates connection between sender and receiver using port numbers
- Ensures receiver is not overrun with data (end to end flow control)
- Perform end to end correction (checksum)
- Allows sending of high priority data

### UDP
- Connectionless
- A transport layer protocol used in place of TCP
