## Today's Takeaways: Form & Validation, Regex, DOM, JavaScript, Game Logic

### Form & Validation
- Check empty first before regex — empty string fails regex and shows wrong error
- & in HTML must be escaped as &amp;
- onsubmit="return validateForm()" — store both validation results in variables first, then && them to ensure both run
- Typos in function names silently break everything — validateFornm vs validateForm

### Regex
- \s uses backslash, /s is wrong
- \ before ^ and $ outside [ ] is wrong — they don't need escaping there
- new RegExp(variable, "g") builds regex dynamically from user input — can't use /variable/g syntax

### DOM
- Form inputs/textarea → .value
- Spans/divs/paragraphs → .innerHTML
- disabled only works on form elements, not <span> — use onclick = null instead
- onclick = overwrites, unlike addEventListener which stacks
- document.createElement("li") + appendChild() dynamically adds to a <ul>

### JavaScript
- while (array.length < 3) not while (array < 3)
- matches.length is a property, not a function — no ()
- parseInt() needed before doing math on .innerHTML values
- setInterval(fn, 2000) — pass function reference, no (), store in global variable for clearInterval
- this.innerHTML inside onclick passes the clicked element's content into a function

### Game Logic
- Declare timer globally so both startGame() and stopGame() can access it
- Disable chars after one click in charMatch(), re-enable in getRanChars() after new chars generated
