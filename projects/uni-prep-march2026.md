## Key Python Concepts to Refresh (Common Pitfalls)

1.  **Mutability vs. Immutability:** Understanding which data types are mutable (lists, dicts, sets - can be changed in place) and immutable (strings, tuples, numbers - require new objects for changes). This impacts how you pass and modify data.
2.  **Scope (LEGB Rule):** Local, Enclosing function locals, Global, Built-in. How variables are accessed and modified within functions.
3.  **List Comprehensions:** A concise way to create lists. Often underutilized but very powerful for cleaner code (e.g., `[x**2 for x in range(10) if x % 2 == 0]`).

## Web Dev Starter Map: HTML, CSS, JS Connection

*   **HTML (HyperText Markup Language): The Structure (Nouns)**
    *   **What it is:** The backbone of every webpage. It defines the content and its basic structure.
    *   **Analogy:** Imagine the steel frame, concrete, and walls of a building.
    *   **Example:** A `<h1>` tag means "this is a main heading", a `<p>` tag means "this is a paragraph".

*   **CSS (Cascading Style Sheets): The Style (Adjectives)**
    *   **What it is:** Describes how HTML elements should be displayed on screen. It controls colors, fonts, layout, spacing, etc.
    *   **Analogy:** The paint colors, wallpaper, furniture, and landscaping of the building.
    *   **Example:** Making all `<h1>` tags red and centered.

*   **JavaScript (JS): The Interactivity (Verbs)**
    *   **What it is:** A programming language that enables dynamic and interactive features on a webpage.
    *   **Analogy:** The electrical system, elevators, and automatic doors that make the building functional and responsive.
    *   **Example:** A button that changes an image when clicked, a form that validates input.

**How they connect:** HTML provides the content. CSS makes it look good. JavaScript makes it behave dynamically. They work together in the web browser to render the complete interactive experience.
