# Searching with Time
### Timestamp
- When logs are ingested, timestamp is stored in _time field
- It is a default field similar to host, source and sourcetype
- Expressed in Unix or epoch time and translated to human readable

### Custom time searching under Advanced

### Default Time Fields
- Events with Timestamp info have date_* fields i.e. date_hour, date_month, date_year
- These fields are generated for events that include date/timestamp in the raw data
- These fields do not change based on a user’s time zone
- If the time zone is specified from the user, the Time column will reflect local time converted from the timestamp in the logs
- If events do not include date/timestamp, the time column will default to the index time or the time these events were indexed in Splunk

### Bin Command used with _time field

- Limit is used to limit the columns by a specified value in this case, we are telling it to limit by 5 columns.
